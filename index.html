Database Views: Virtual Windows

A View is a virtual table based on the result-set of an SQL statement. It doesn't store data itself; it just looks at the base table. Crucially, for simple views, updating the view directly updates the underlying base table.

Base Table: Students

Triggers: Automated Actions

A Trigger is SQL code that automatically runs when an event (INSERT, UPDATE, DELETE) occurs. In this Hospital Billing scenario, we have a trigger that automatically calculates the DueAmount whenever a new payment (PaidAmount) is updated.

-- The hidden Trigger logic executing in background

CREATE TRIGGER UpdateDue ON Billing

AFTER UPDATE AS

BEGIN

UPDATE Billing

SET DueAmount = TotalAmount - PaidAmount

WHERE PID IN (SELECT PID FROM inserted);

END;
